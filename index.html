Vue.js Basic Examples

1. Hello World

To create a Vue instance, pass an object to the constructor like so. The "data" object represents the instance's internal state. The el property tells Vue which element on the page it should take over (or mount to).

Properties on the data object can be treated as if they were defined directly on the parent. When data changes in the app, any necessary updates to the DOM which it controls happen instantly and automatically. There is *no* need to do direct DOM manipulation here. 

          
var app = new Vue({
  el: '#app1',
  data: {
    message: 'Hello World!'
  }
})

app.message = 'Here is an updated message.'
          
          
<div id="app1">
{{ message }}
</div>

Counter App

This example shows how a Vue instance's internal state can be updated in response to various events. Because of Vue's reactive nature, as soon as any internal data is updated, any elements that depend on that data update automatically - handling events becomes much simpler

In addition to data, a Vue instance can have methods - these are just functions attached to the object. Any property in the objects' data can be accessed with this + the property name

Elements inside a Vue template can have event handlers defined on them as attributes using hte v-on directive. Ex: <button v-on:click="someMethod">

This can be abbreviated as @click="someMethod" for many common events.

          
var counter = new Vue({
  el: '#app2',
  data: {
    count: 0
  },
  methods: {
    increment: function () {
      this.count ++
    },
    decrement: function () {
      this.count --
    }
  }
})
          
          
<div id="app2">
  <h4>The current count is: {{ count }}</h4>
  <button @click="increment">+</button>
  <button @click="decrement">-</button>
</div>

User Input & Computed Properties

In addition to methods, Vue instances support computed properties. These properties are automatically derived from some other source (often something in the instance's state), and any arbitrary transformation can be applied. As soon as the source data changes, the computed property changes as well.

The template for this instance contains a special directive, v-model, which is defined on an <input> element. This means that the value of the input element is tied to the relevant property of this instance (in this case, message). v-model and related tools make handling user input very easy in Vue.

          
var reverser = new Vue({
  el: '#app3',
  data: {
    message: ''
  },
  computed: {
    reversedMessage: function () {
      return this.message.split('').reverse().join('')
    }
  }
})
          
          
<div id="app3">
  <h4>Reverse-o-matic!</h4>
  <input v-model="message" />
  <p>Type a message above and see it reversed here:</p>
  <p>{{ reversedMessage }}</p>
</div>
